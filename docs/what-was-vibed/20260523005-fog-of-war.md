# Fog of war

Implements `docs/game-design.md §6` end-to-end. Each player only sees the
grid through a per-player view that's frozen to "what they last saw"
outside their current vision radius. Enemy tower construction is visible
but the type is hidden until completion; tower upgrades are silent
(memory keeps the old level until the new one is seen post-build); enemy
creep upgrade research is invisible (sidebar already only shows the
planning player's own upgrades, so no work there). Creep corpses persist
across the sim/plan boundary for inspection.

## What changed

- **Vision is a configurable per-level / per-upgrade stat.** Added
  `vision <N>` to `TowerLevelStats` (in `data/towers.cfg` and parsed by
  `tower_config.c`) and to `CreepUpgradeConfig` with a new
  `CREEP_UPG_SET_VISION` merge flag. The merge in
  `compute_active_profiles` overlays it like every other field — later
  completed upgrades setting `vision` win.
- **`PlayerView` per player on `GameState`.** Holds three things:
  - `vis_now[W][H]` — current-frame Chebyshev union of every alive,
    fully-built tower's and every alive creep's vision radius.
  - `ever_seen[W][H]` — sticky union; never cleared except by `game_init`.
  - `mem[W][H]` — last-seen `FogTowerMemory` snapshot per cell.
  - `creep_mem[MAX_THINGS]` — last-seen `FogCreepMemory` snapshot per
    thing index. Wiped only at `start_simulation` so corpses outlast the
    sim.
- **`fog_refresh(viewer)`** recomputes the above and is called for both
  players at `game_init_state`, after any `try_place` /
  `game_upgrade_selected` / `game_destroy_selected`, at
  `start_simulation`, at the end of every `sim_one_tick`, at end of
  `end_simulation` (so build-turn completions take effect for the next
  plan), and at the tail of `game_snapshot_load`.
- **Memory rules for enemy towers** (only piece with subtle behaviour):
  | Live tower at visible cell                | mem update                              |
  |---|---|
  | Own tower (any state)                     | full snapshot                           |
  | Enemy, fully built                        | full snapshot                           |
  | Enemy, initial build (build_turns>0, L1)  | `FOG_OCC_BUILDING` (type withheld)      |
  | Enemy, mid-upgrade (build_turns>0, L>1)   | leave mem unchanged — silent upgrade    |
  | Empty visible cell                        | `FOG_OCC_NONE`                          |
- **Render layer rewrites three loops.** `render.c::render_frame` sets a
  module-static `g_viewer` from phase + `sim_viewer`, then:
  - Towers: single-pass over the grid; if a cell is in vis_now show
    live state (with the building/upgrade caveats), else fall back to
    `mem[x][y]` (or nothing). Out-of-vision known towers render dimmed.
  - Beams: hidden when the source isn't visible to the viewer.
  - Creeps: own always shown; enemy filtered by vis_now. A second pass
    over `creep_mem` paints "corpses" (hollow circles) for entries the
    live loop didn't already cover.
  - Sidebar: enemy player's `$N +N` line is masked to `$??? +??`.
  - Zones, debris, spawn / flag / receptacle landmarks are drawn at
    full strength regardless of fog — per §6 the "empty map" is always
    visible; fog only hides entities. The absence of an entity draw at
    a cell is the only signal of frozen / unknown state.
- **Viewer choice happens before the sim runs, not during.** A new
  `PHASE_PRE_SIM` sits between `PLAN_BLUE` and `SIMULATE`: both
  lock-ins now land in PRE_SIM, where the sidebar offers two committal
  buttons — `BTN_START_SIM_AS_RED` and `BTN_START_SIM_AS_BLUE`. Each
  calls `game_choose_sim_view(viewer)` which sets `sim_viewer` and
  fires `start_simulation()`. SIMULATE has no controls (just a "View:
  RED/BLUE" label so you know whose lens you're in). Snapshot phase
  char `V` round-trips PRE_SIM.
- **Catalog content.** `data/towers.cfg` now carries `vision` per
  level (Blocker 1, Gunner 3/5/7, Splasher 3/4, Resource 2).
  `data/creep_upgrades.cfg` gets vision values for the existing trees
  (Swarm/Attacker/Tank baselines = 2) and adds a new SCOUT creep type
  with a single `SCOUT_1` upgrade granting vision=6, satisfying the
  baseline scout role from §8 of the design doc. (The follow-on scout
  upgrades — reveal in-progress upgrades, post-death vision — are out
  of scope for this change.)

## What didn't change

- **Sim/pathing semantics.** Per §10 of the design doc, creeps
  deliberately "know" things the player doesn't. The BFS in `game.c`
  reads the truth; fog is purely a render input.
- **Snapshot format.** `game_snapshot_encode/load` is unchanged. The
  view state (`vis_now`, `ever_seen`, `mem`, `creep_mem`) is rebuilt
  from current truth at the tail of `game_snapshot_load` via
  `fog_refresh_all()`. Reloading a snapshot intentionally doesn't
  resurrect a specific viewer's history.

## Tests added (one at each layer it belongs to)

- `tests/test_tower_config.c::test_vision_field` — parser sets the
  per-level int; absent → 0.
- `tests/test_creep_config.c::test_vision_field` — parser sets the
  field and flips `CREEP_UPG_SET_VISION`; absent → 0 with the flag clear.
- `tests/test_game.c` — seven game-layer fog cases:
  - `test_fog_spawn_baseline` — own spawn cell is always visible to its owner.
  - `test_fog_tower_extends_vision` — placing a tower with vision=2
    illuminates a 5×5 Chebyshev box; enemy view is untouched.
  - `test_fog_building_obscures_type` — enemy under construction in
    view → `mem.occ == FOG_OCC_BUILDING`, no type.
  - `test_fog_completed_tower_visible` — after the build completes
    in view, mem upgrades to `FOG_OCC_TOWER` with the correct type/level.
  - `test_fog_upgrade_is_silent` — starting an upgrade while in view
    does NOT advance the viewer's remembered level.
  - `test_fog_creep_memory_survives_sim_end` — a creep memory entry
    is still `valid` after `end_simulation`'s mass death.
  - `test_fog_sim_viewer_toggle` — `game_toggle_sim_viewer` flips
    `sim_viewer`.
- `tests/test_render.c` — two new cases (plus an updated
  `simulation_phase_no_buttons` to assert the toggle is hittable):
  - `test_fog_hides_enemy_tower_glyph` — under fog, the enemy
    Gunner's `"G1"` text never appears at its cell.
  - `test_fog_masks_enemy_sidebar_resources` — viewer's own line
    shows `$100`, opponent's shows `$???`.

## Footprint

`GameState` grows by `2 × sizeof(PlayerView)` ≈ 30 KB (per-cell mem +
ever_seen + vis_now + per-thing creep_mem). Total mutable static is now
≈ 88 KB, still well under L1. Zero heap allocations preserved.

## Things deliberately deferred

- Scout creep follow-on upgrades from §8: "reveal enemy tower upgrades
  in progress" and "vision persists 2 turns after corpse". Both can be
  added later as additional `CreepUpgradeConfig` fields with new merge
  flags — no architectural change needed.
- The "default creep pathing is always visible" sub-clause from §6 — I
  treat zones / landmarks as the "always visible" base. The default
  pathing line can be reintroduced cheaply by pre-baking `ever_seen` for
  the cells of the no-tower BFS path during `game_init_state`.
